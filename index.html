JAVA METHOD NAMING

方法命名的艺术

一个好的方法名应清晰表达其功能、返回值和副作用

获取型: getUser(), fetchDetails()
查询型: findById(), searchByName()
判断型: isValid(), hasPermission()
get前缀表示一定能返回结果，find表示可能返回null
方法命名反面教材

危险的方法命名

常见方法命名错误：

  • process() - 过于宽泛，不表达意图
  • getData() - 不明确获取什么数据
  • check() - 不清楚检查什么，返回什么
方法名误导性可能导致严重bug，如名为check却修改数据
变量命名规则

变量类型与命名规则

1
局部变量: camelCase orderCount, userName
2
常量: UPPER_SNAKE_CASE MAX_RETRY_COUNT, DEFAULT_TIMEOUT
3
集合变量: 使用复数名词 List<User> users (而非 userList)
只有真正不变的静态final值才应使用全大写命名
集合命名技巧

集合变量命名
最佳实践

List/数组: 使用复数名词 - users
Map: 复数名词+By+键特性 - usersByName
Set: 独特性+复数名词 - uniqueNames

避免在名称中包含类型信息，如userList
